\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Comparable, class Compare = std\+::less$<$\+Comparable$>$$>$\newline
class Queue\+::\+Binary\+Heap$<$ Comparable, Compare $>$}

Priority queue. 

Priority queues are a type of container adaptors, specifically designed such that its first element is always the greatest of the elements it contains, according to some strict weak ordering criterion.\textbackslash{} This context is similar to a heap, where elements can be inserted at any moment, and only the max heap element can be retrieved (the one at the top in the priority queue).


\begin{DoxyTemplParams}{Template Parameters}
{\em Comparable} & Type of the elements.\textbackslash{} Aliased as member type priority\+\_\+queue\+::value\+\_\+type.\\
\hline
{\em std\+::less$<$\+Comparable$>$} & A binary predicate that takes two elements (of type T) as arguments and returns a bool.\textbackslash{} The expression comp(a,b), where comp is an object of this type and a and b are elements in the container, shall return true if a is considered to go before b in the strict weak ordering the function defines.\textbackslash{} The priority\+\_\+queue uses this function to maintain the elements sorted in a way that preserves heap properties (i.\+e., that the element popped is the last according to this strict weak ordering).\textbackslash{} This can be a function pointer or a function object, and defaults to less$<$\+T$>$, which returns the same as applying the less-\/than operator (a$<$b).\textbackslash{} 即：comp(a, b)返回true，当且仅当b的优先级高于a. 故若comp为less，则为最大堆. \\
